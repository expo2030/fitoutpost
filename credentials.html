§1 · Overview

What FitOut Post Is

FitOut Post is a self-contained, server-free news intelligence platform for the global fit-out and interior construction industry. It covers fit-out (also written fitout or fit out) — the commercial interior works that transform raw building shells into finished offices, hotels, retail spaces, hospitals, and cultural venues.

Every page is a static HTML file that runs directly from a local folder (file://), with no backend, no hosting costs, and no external dependencies at runtime. Data is embedded as JSON inside each page and refreshed by running Python build scripts on demand.

Business rationale: The platform aggregates industry intelligence that industry professionals currently have to hunt across dozens of sources. The free news layer builds audience; the tenders paywall creates a monetisation route for high-intent users (contractors, consultants, suppliers) who need procurement intelligence.

§4 · Sources

All Data Sources Used

News Feed (fetch_news.py)

Two complementary source types are combined and deduplicated on each run.

Google News RSS Queries — 395 targeted searches

The fetcher uses news.google.com/rss/search to run 395 geo-targeted query strings. Queries cover the three spelling variants (fitout, fit-out, fit out) combined with sector terms (office, retail, hotel, healthcare, museum, stadium), contractor terms, and contract types. Region/language parameters are set to bias results by geography (UK, UAE, AU, IN, DE, FR, ES, MX, etc.).

Direct RSS Feeds — 30 trade publications

Interiors Monthly (UK) Contractor Magazine (AU) MEED (ME) Contract Journal (UK) BDC Network (US) Interiors & Sources (US) Commercial Design (IN) Commercial Interior Design (ME) Construction World (IN) CNBC Africa Building Design & Construction (US) Dezeen Designboom Wallpaper* Interior Design (US) Elle Decor Architectural Digest (US) Architectural Digest (IN) AD Magazin (DE) Sleeper Media Surface Mag Azure Magazine Icon Eye Metropolis Mag Design Week (UK) Hospitality Design Indesign Live (AU) Living Etc House Beautiful Design Milk

Quality filter: Every article must pass a two-regex gate — MUST_HAVE (contains a fit-out synonym) and a loose relevance check — before it enters news.json. Articles are geo-detected by scanning title and description against a country/continent reference table of 100+ place names.

Project Pipeline (fetch_pipeline.py)

The pipeline aggregator runs ~200+ Google News RSS queries specifically designed to surface project announcements, planning permissions, development signings, and construction starts — without using the word "tender." Queries target project announcement language ("to open", "underway", "breaks ground", "to deliver", "signs", "awarded to") combined with sector and geography signals.

Each article is classified into one of 9 sectors (Hospitality, Commercial, Cultural & Museums, Sports & Leisure, Retail, Infrastructure, Education, Healthcare, Offices) using keyword matching. Continent and country are geo-tagged from article text.

§5 · Architecture

Build Pipeline & Technical Architecture

All pages follow the same pattern: a Python fetch script pulls raw data from APIs and RSS feeds, cleans and deduplicates it, and saves it to a JSON file. A central build.py script reads each JSON file, embeds it as a <script type="application/json"> block inside the corresponding HTML template, and writes the output page.

Fetch scriptData fileTemplateOutput page
fetch_news.pynews.json_template.htmlindex.html, site.html
fetch_pipeline.pypipeline.json_pipeline_template.htmlpipeline.html
fetch_tenders.pytenders.json_tenders_template.htmltenders.html
fetch_intelligence.pyintelligence.json_intelligence_template.htmlintelligence.html
fetch_weekly.pyweekly.json_weekly_template.htmlweekly.html
fetch_pipeline_updates.pypipeline_updates.json→ merged by build.pypipeline.html

Key Design Decisions

No server required. All pages open directly from a local folder using the file:// protocol. Data is baked into each HTML file as embedded JSON. This means zero hosting costs, instant page loads, and no API keys exposed at runtime.

Deduplication. Each fetch script loads the existing JSON before fetching, and skips any article whose ID (MD5 of the URL) already exists. This means re-running a fetch only adds new articles.

Paywall. The tenders page implements a client-side email gate using localStorage. Contract value, deadline, and CPV code fields are CSS-blurred until an email address is entered. No server verification — this is a frictionless soft gate for audience building.

Project tracking. Pipeline projects can be bookmarked in the browser via localStorage. The tracked IDs are exported to JSON, fed into fetch_pipeline_updates.py, and the resulting update timelines are embedded in the next pipeline build.

Admin tool. admin.py runs a local Flask server on localhost:5050 providing a simple UI to manage the editorial sections (α Edge articles, β Edge polls, γ Edge games) without editing JSON by hand.
